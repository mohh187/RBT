# تشغيل الدومينات والإرسال باسم المنشأة — دليل المالك

بُني في 2026-07-18. كل الكود جاهز؛ هذه الخطوات تُفعّله على البنية التحتية.

## 1) السب-دومين التلقائي لكل منشأة (slug.rbt360sa.com)

الكود يحلّ `اسم-المنشأة.rbt360sa.com` تلقائياً من `tenantSlugs` — كل منشأة تحصل على
رابطها فور التسجيل بلا أي إعداد. المطلوب منك مرة واحدة على Cloudflare:

### أ. سجل DNS شامل (wildcard)
- DNS → Records → أضف: `CNAME` باسم `*` يشير إلى `rbt360sa.com` — **Proxied (سحابة برتقالية)**.
  (عكس سجلات الجذر التي يجب أن تبقى DNS-only لشهادة Firebase.)

### ب. Cloudflare Worker وسيط (لأن Firebase Hosting لا يدعم wildcard)
- Workers & Pages → Create Worker باسم `venue-subdomains` والصق:

```js
export default {
  async fetch(request) {
    const url = new URL(request.url)
    // اخدم التطبيق نفسه من أصل Firebase؛ المتصفح يبقى على السب-دومين
    // والتطبيق يقرأ location.hostname ويحلّ المنشأة بنفسه.
    url.hostname = 'menu-88996.web.app'
    return fetch(new Request(url, request))
  },
}
```

- Worker → Triggers → Routes → أضف Route: `*.rbt360sa.com/*` (منطقة rbt360sa.com).
- استثنِ الجذر وwww إن لزم (اتركهما على توجيه DNS-only الحالي — الـ Route يلتقط
  السب-دومينات فقط ما دام الجذر بلا سجل proxied).
- SSL/TLS: وضع **Full** + شهادة Cloudflare Universal تغطي `*.rbt360sa.com` تلقائياً.

اختبار: افتح `اسم-منشأتك.rbt360sa.com` → يجب أن يفتح منيو المنشأة مباشرة.

## 2) الدومين الخاص بمنشأة (مثل menu.cafex.com)

- المنشأة تطلبه من: الإعدادات ← التكاملات ← الربط الفني ← «النطاق المخصّص»
  (يظهر لصاحب صلاحية الإدارة/التكاملات) — يُنشئ طلباً «قيد التفعيل».
- صاحب المنشأة يضيف عند مسجّله: `CNAME` من نطاقه إلى `connect.rbt360sa.com`
  (زر «فحص DNS» في البطاقة يتحقق فعلياً عبر DNS-over-HTTPS).
- أنت من كونسول المنصة (/platform → Domains) تضغط تفعيل بعد جاهزية DNS:
  - أضف الدومين في Firebase Hosting (Add custom domain) أو فعّل Cloudflare for SaaS
    (SSL for SaaS → Custom Hostnames) إن كثرت الدومينات.
  - ثم غيّر حالة الوثيقة في `domains/{host}` إلى `active`.

## 3) الإرسال باسم المنشأة

### واتساب
- **الوضع الافتراضي (جاهز):** رقم المنصة يرسل، وكل رسالة تبدأ باسم المنشأة
  (القوالب والنصوص كلها تحمل {المنشأة}).
- **رقم المنشأة الخاص (جديد):** صفحة الحملات ← «أرسل برقم واتساب منشأتك الخاص»
  (تظهر لصاحب صلاحية التكاملات): تضع المنشأة Phone Number ID + توكن System User
  الدائم من حساب Meta الخاص بها + أسماء قوالبها المعتمدة. تُخزَّن في
  `tenants/{tid}/private/wa` (غير عامة أبداً — قواعد Firestore تحصرها بالتكاملات).
  كل مسارات الإرسال (حالة الطلب، الفاتورة، الحملات، الأتمتة، الترحيب، الترقية،
  عيد الميلاد، اشتقنا لك، ما بعد الزيارة، تقرير المالك) تستخدم رقمها تلقائياً فور الحفظ.

### البريد (Resend)
- العنوان يبقى على دومين المنصة الموثّق (شرط Resend)، والاسم الظاهر يصبح
  «اسم المنشأة عبر rbt360» تلقائياً، وأي ردّ يذهب إلى بريد المنشأة (contactEmail).
- فعّل Resend كما في HOSTING_EMAIL_WHATSAPP_PLAN.md (سجلات SPF/DKIM على Cloudflare
  بوضع DNS-only + RESEND_API_KEY + EMAIL_FROM في functions/.env ثم نشر الدوال).

## 4) الفواتير للعملاء

- **واتساب:** رابط الفاتورة الضريبية يصل تلقائياً عند الدفع (موجود سابقاً + الآن برقم المنشأة إن وُصل).
- **بريد (جديد):** حقل بريد اختياري في سلة الطلب («لاستلام الفاتورة») — إن عبّأه العميل
  تصله الفاتورة بريدياً باسم المنشأة مع رابط العرض، إضافة إلى واتساب.

## 5) قوالب الرسائل المخصصة (جديد)

صفحة الحملات ← «قوالب الرسائل باسم منشأتك»: 8 قوالب (حالة الطلب، الفاتورة، الترحيب،
الترقية، عيد الميلاد، عرض جديد، صنف مميز، أصناف جديدة) بمتغيرات
{الاسم} {المنشأة} {الصنف} {الكود} {الطلب} {الحالة} {المبلغ} {الرابط} {العضوية}.
الفارغ منها يستخدم الصياغة الافتراضية. (رسالتا «اشتقنا لك» و«ما بعد الزيارة» لهما
حقلاهما الخاصان في لوحة الأتمتة نفسها.)

## 6) CORS للتخزين (إلزامي لميزات الصور بالذكاء)

المتصفح يمنع قراءة صور Firebase Storage من الكود (قصّ الصور، توليد الصور بمرجع صنف)
حتى تضبط CORS على الحاوية **مرة واحدة**. الملف جاهز في جذر المشروع `cors.json`:

```
gcloud storage buckets update gs://menu-88996.firebasestorage.app --cors-file=cors.json
```
(أو بالأداة القديمة: `gsutil cors set cors.json gs://menu-88996.appspot.com` —
تحقق من اسم الحاوية الفعلي في Firebase Console ← Storage.)

ملاحظة: حتى قبل ضبطه، «رفع المراجع مباشرة» في استوديو المنشورات ومصمم الشرائح
يعمل دائماً (ملف محلي لا يمر بـ CORS)، والتوليد بمرجع صنف يسقط تلقائياً لمشهد حر
بدل الفشل.
