# ربط واتساب (Meta Cloud API) — دليل خطوة بخطوة

الكود جاهز ومنشور في `functions/messaging.js` (`sendWhatsAppTemplate` + مُشغّل
`onOrderCustomerNotify`) و`functions/invoicing.js` (رسالة رابط الفاتورة). يعمل «فشلاً
آمناً» طالما القيم فارغة — أي لا يرسل شيئاً حتى تملأ بيانات الاعتماد. المطلوب منك:
تعبئة قيمتين + اعتماد قالب رسالة، ثم إعادة نشر الدوال.

القيم المطلوبة في `functions/.env` (اثنتان فارغتان الآن):
```
WA_PHONE_NUMBER_ID=      ← تملؤها من لوحة Meta
WA_ACCESS_TOKEN=         ← توكن دائم من System User
WA_LANG=ar               (مضبوط)
WA_TEMPLATE_ORDER_UPDATE=order_update   (مضبوط)
WA_TEMPLATE_RECEIPT=     (اختياري — رسالة رابط الفاتورة)
```

---

## 1) حساب Meta Business
- افتح business.facebook.com → أنشئ «Business Portfolio» باسم نشاطك وبريدك.

## 2) تطبيق مطوّر + منتج WhatsApp
- افتح developers.facebook.com → My Apps → **Create App** → النوع **Business** → اربطه بالـ Portfolio.
- من صفحة التطبيق: **Add Product** → اختر **WhatsApp** → Set up.

## 3) احصل على Phone Number ID
- داخل التطبيق: **WhatsApp → API Setup**. سترى:
  - **Phone number ID** ← انسخه إلى `WA_PHONE_NUMBER_ID`.
  - **WhatsApp Business Account (WABA) ID** ← ستحتاجه للقوالب.
  - **رقم تجريبي مجاني** يعطيك إياه Meta (يكفي للاختبار).
  - **Temporary access token** (صالح 24 ساعة فقط — لا تستخدمه للإنتاج، انتقل للخطوة 4).
- في نفس الصفحة: أضِف رقمك الشخصي في «To» (يسمح بحتى 5 أرقام مستقبِلة للاختبار مع الرقم التجريبي).

## 4) توكن دائم (مهم — توكن اللوحة ينتهي خلال 24 ساعة)
- افتح business.facebook.com/settings → **Users → System Users** → **Add** → أنشئ System User (صلاحية Admin).
- **Assign assets** → **Apps** → اختر تطبيقك → **Full control**.
- اضغط **Generate new token** → اختر التطبيق → فعّل الصلاحيتين:
  - `whatsapp_business_messaging`
  - `whatsapp_business_management`
- **Generate** → انسخ التوكن (يظهر مرّة واحدة) إلى `WA_ACCESS_TOKEN`.

## 5) أنشئ قالب الرسالة واعتمده (order_update)
الرسائل التي تبدأها المنشأة (خارج نافذة 24 ساعة) **يجب** أن تمرّ عبر قالب Utility معتمد.
- افتح business.facebook.com/wa/manage → **Message templates** → **Create template**.
- الفئة: **Utility**.
- الاسم: `order_update` (يجب أن يطابق `WA_TEMPLATE_ORDER_UPDATE`).
- اللغة: **العربية**.
- **جسم الرسالة بثلاثة متغيّرات** بالترتيب: `{{1}}` اسم المنشأة، `{{2}}` رقم الطلب، `{{3}}` حالة الطلب. مثال:
  ```
  عميلنا العزيز، {{3}}.
  المنشأة: {{1}}
  رقم الطلب: {{2}}
  شكراً لك.
  ```
  عيّنات المتغيّرات المطلوبة عند الإنشاء: {{1}} = مقهى مزاج، {{2}} = #1234، {{3}} = طلبك جاهز.
- **أرسِل للمراجعة**. الاعتماد عادةً من دقائق إلى ساعات.

### (اختياري) قالب رابط الفاتورة `receipt`
- قالب Utility ثانٍ باسم `receipt` بأربعة متغيّرات: `{{1}}` المنشأة، `{{2}}` رقم الطلب، `{{3}}` الإجمالي، `{{4}}` رابط الفاتورة. مثال:
  ```
  {{1}}: تم استلام دفعتك للطلب {{2}} بمبلغ {{3}} ريال.
  فاتورتك: {{4}}
  ```
- ثم اضبط `WA_TEMPLATE_RECEIPT=receipt`. إن تركته فارغاً يُرسل النظام نصاً حرّاً (يعمل داخل نافذة 24 ساعة فقط).

## 6) املأ `functions/.env` وأعد النشر
```
WA_PHONE_NUMBER_ID=<من الخطوة 3>
WA_ACCESS_TOKEN=<التوكن الدائم من الخطوة 4>
```
ثم (الدوال v2 تخبز env عند النشر):
```
firebase deploy --only functions:onOrderCustomerNotify,functions:onOrderPaid
```
(أو `firebase deploy --only functions` لإعادة نشر الكل.)

## 7) الاختبار
- تأكّد أن رقمك مضاف في قائمة المستقبِلين التجريبيين (الخطوة 3).
- من لوحة الإدارة غيّر حالة طلب (قبول/تحضير/جاهز) → يصل واتساب لرقم العميل.
- ادفع طلباً أونلاين → تصل رسالة «رابط الفاتورة».

## 8) الانتقال للإنتاج (عملاء حقيقيون)
- الرقم التجريبي محدود بـ 5 مستقبِلين. لمراسلة عملاء حقيقيين:
  - أضِف **رقم جوال حقيقي** (غير مستخدَم في واتساب شخصي) إلى WABA وفعّله برمز.
  - أكمل **توثيق النشاط (Business Verification)** في Meta.
  - يبدأ رقمك بحدّ 1,000 محادثة/يوم ويرتفع (10K ← بلا حدّ) حسب جودة الرسائل.

## ملاحظات
- **لا حاجة لـ webhook** لإرسال تحديثات الطلبات — الـ webhook فقط لاستقبال ردود العملاء/حالات التسليم (اختياري لاحقاً).
- تحكّم المنشأة: مفتاح «إشعارات العميل» في الإعدادات (`customerNotify.whatsapp`) يوقف الإرسال لتلك المنشأة.
- الموافقة (PDPL): إعطاء العميل رقمه عند الطلب = موافقة ضمنية على تحديثات الطلب. يُستحسن إضافة ملاحظة «للإيقاف أرسِل …».
- تكلفة: Meta تحتسب لكل «محادثة Utility» بعد الحصّة المجانية الشهرية.
