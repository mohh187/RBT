# استوديو شاشات العرض — محرر/مصمم/متحكم كامل (خطة مؤجلة — نعود لها بعد إنهاء الخطط السابقة)

> الوضع الحالي (منفذ): شاشات باقتران رمز + بلاي ليست (صور/فيديو/شرائح منيو حية) + إشعارات «طلبك جاهز» فوق المحتوى. هذه الخطة = الطبقة الاحترافية فوقه.

## البنية المقترحة
- **محرر كانفس** `/admin/screens/designer/{slideId}`: طبقات (خلفية لون/صورة/فيديو + نصوص + صور + أشكال + QR + عناصر منيو حية) بسحب وتغيير حجم ودوران؛ تُخزن كـ JSON schema `{layers:[{type,x,y,w,h,rot,style,binding?}]}` على شريحة من نوع جديد `design` — **المشغّل يرسمها مباشرة** (بلا تصدير صور) فتبقى حية وقابلة للربط بالبيانات.
- **الربط الحي (bindings)**: عنصر نص يربط بـ {اسم صنف/سعره/عرض نشط/كود خصم/عدّاد} فيتحدث تلقائياً من Firestore.
- **معاينة حية**: iframe لنفس `/screen` بوضع معاينة (نفس آلية StaffPreview) + محاكاة مقاسات (FHD أفقي/عمودي/4K).
- **التحكم بالأحجام وطريقة العرض**: لكل شاشة `orientation` و`resolution` و`fit`؛ الشرائح تصمم على شبكة نسبية (٪) فتتكيف مع أي شاشة.

## مراحل التنفيذ عند العودة
S1 المحرر الأساسي (طبقات نص/صورة/شكل + حفظ design slide + رسمها في المشغّل) → S2 القوالب والربط الحي (bindings + عرض اليوم + QR) → S3 الحركة والجدولة والريموت → S4 المجموعات والتحليلات والباقات.
