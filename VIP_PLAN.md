# خطة عضوية VIP / بطاقة الولاء — MenuLink

> هدف: نظام عضوية متكامل ببطاقة قابلة للمسح، نقاط تتراكم مع كل زيارة، خصومات وامتيازات دائمة، وإضافة للمحفظة (Apple/Google Wallet) — **دون أي تعارض** مع نظام الولاء الحالي.

---

## 1) المبدأ الأساسي (عدم التعارض)

النظام الحالي = **بطاقة مشروبات مثقّبة**: كل `loyaltyThreshold` مشروب ← مكافأة مشروب مجاني (`rewards`). هذا يبقى **كما هو، لا يُمَس**.

عضوية VIP = **طبقة فوقه** بعملة منفصلة:

| العملة | الحقل | المعنى | المصدر |
|--------|-------|--------|--------|
| مكافآت المشروبات | `rewards` (موجود) | مشروب مجاني كل N مشروبات | تلقائي عند الطلب |
| **نقاط العضوية** | `membership.points` (جديد) | تحدّد المستوى وتُستبدل بخصم | تلقائي عند **الدفع** |

> قاعدة ذهبية: النقاط تُكتسب **مرة واحدة لكل طلب مدفوع** (ربط بـ`orderId`) لمنع التكرار/التلاعب.

---

## 2) آلية البطاقة: لمن ومتى وكيف

**لمن تُعطى:**
- **منح تلقائي**: عند تحقّق `membershipPolicy` (قابلة للضبط):
  - `minOrders` (افتراضي 10) **و** `minSpent` (افتراضي مبلغ) **و** `minAvgBasket` (متوسط سلة جيد).
  - النظام يفحص العميل بعد كل طلب؛ إن تحقّقت الشروط ولم يكن عضواً ← يُمنح تلقائياً + إشعار ترحيب.
- **منح يدوي**: زر «امنح عضوية» في بطاقة العميل لموظف مخوّل (لعميل مميّز استثنائي).

**متى:** لحظة استيفاء الشروط (تلقائي)، أو بقرار الموظف (يدوي).

**كيف تُمسح:**
- لكل عضو `memberId` مقروء (`NEM-7F3K`) + `token` غير قابل للتخمين.
- QR يُشفّر `…/mcard/{slug}/{token}` → يفتح بطاقة العضو.
- **ماسح الموظف** (توسعة Scanner الحالي) يقرأ الـtoken → يفتح بطاقة العضو ← منح/خصم نقاط ومعرفة السجل كاملاً.

---

## 3) المستويات والامتيازات

| المستوى | العتبة (نقاط/إنفاق) | خصم دائم | امتيازات |
|---------|---------------------|----------|----------|
| 🥈 فضي | عضو جديد | 5% | إشعارات أولوية |
| 🥇 ذهبي | متوسط | 10% | + عروض حصرية + أولوية حجز |
| 💎 بلاتيني | عالٍ | 15% | + هدايا دورية + دعوات فعاليات |

- خصم المستوى **يُطبَّق تلقائياً** على طلب العضو (عبر حقل `loyaltyDiscount` الموجود — لا حقل جديد في الطلب).
- النقاط **تُستبدل اختيارياً** بخصم إضافي (مثال: 100 نقطة = 5 ريال).

---

## 4) نموذج البيانات

على مستند العميل `tenants/{tid}/customers/{phoneId}`:
```js
membership: {
  active: true,
  tier: 'silver|gold|platinum',
  memberId: 'NEM-7F3K',
  token: '<random>',
  points: 0,            // الرصيد الحالي
  pointsLifetime: 0,    // تراكمي (للترقية)
  pointsRedeemed: 0,
  discountPct: 5,
  source: 'auto|manual',
  joinedAt, lastEarnAt,
  lastEarnOrderId: '',  // منع تكرار الكسب لنفس الطلب
}
```
إعدادات المنشأة `tenants/{tid}`:
```js
membershipPolicy: { enabled, minOrders, minSpent, minAvgBasket, earnRate, redeemRate,
                    tiers: { gold: {...}, platinum: {...} }, pointsExpiryDays }
```

---

## 5) الأسطح

- **بطاقة العميل** (موجودة): تعرض المستوى/النقاط/الخصم المستحق + أزرار منح/خصم/استبدال + السجل الكامل + تقرير.
- **صفحة العملاء** (موجودة): شارة المستوى + فلترة «الأعضاء فقط».
- **ماسح الموظف**: مسح بطاقة عضو → منح نقاط/استبدال.
- **بوابة العضو** (جديدة، عامّة): `…/mcard/{slug}/{token}` تعرض البطاقة + QR + النقاط + السجل + زر «أضف لمحفظة Apple/Google».
- **لوحة تحليلات الولاء** (تقارير): أعضاء نشطون/خاملون، LTV، أثر الخصومات.

---

## 6) Apple / Google Wallet (يتطلب خادماً)

- `.pkpass` لـApple يحتاج **شهادة PassKit + توقيعاً خادمياً** (دالة سحابية Firebase Functions). Google Wallet = JWT موقّع.
- لذلك: مرحلة مستقلة. v1 = بطاقة QR داخل التطبيق؛ Wallet لاحقاً عند توفّر شهادة المطوّر.
- تحديث الرصيد على البطاقة = Wallet push بعد كل كسب.

---

## 7) المراحل

1. **البيانات + المنطق**: حقول `membership` + `membershipPolicy` + دوال `grantMembership / earnPoints(orderId) / redeemPoints / autoEvaluate`.
2. **البطاقة في النظام**: عرض/منح/خصم في بطاقة العميل + شارات صفحة العملاء.
3. **المسح**: توسعة الماسح لقراءة بطاقة العضو.
4. **بوابة العضو العامة** + QR.
5. **الخصم التلقائي** عند الطلب لعضو + العروض الحصرية + الإشعارات الأولوية.
6. **Wallet** (Apple/Google) عبر دالة موقّعة — يتطلب شهادة.
7. **تحليلات الولاء**.

---

## 8) عشرون اقتراحاً

1. بطاقة QR رقمية ثابتة لكل عضو (token آمن) تُمسح من الكاشير أو الماسح.
2. منح تلقائي ذكي: ≥10 طلبات + إنفاق + متوسط سلة جيد (قابل للضبط).
3. منح يدوي بزر في بطاقة العميل لموظف مخوّل (استثناء/مجاملة).
4. مستويات فضي/ذهبي/بلاتيني تترقّى تلقائياً، لكلٍّ امتيازاته.
5. نقاط مع كل طلب مدفوع بمعدّل قابل للضبط، **منفصلة** عن بطاقة المشروب المجاني.
6. خصم دائم تلقائي حسب المستوى يُطبَّق على طلب العضو فوراً.
7. استبدال النقاط بخصم/منتج من الكاشير أو التطبيق.
8. Apple/Google Wallet عبر تمرير موقّع (مرحلة بشهادة مطوّر).
9. تحديث رصيد النقاط حيّاً على بطاقة المحفظة بعد كل زيارة.
10. إشعارات أولوية للأعضاء بالعروض والفعاليات قبل الجمهور.
11. عروض حصرية مرتبطة بفئة العضوية تظهر فقط للأعضاء.
12. أولوية حجز الفعاليات/المقاعد للأعضاء قبل الفتح العام.
13. سجل كامل لطلبات العضو ونقاطه وخصوماته + تقرير قابل للتصدير.
14. مكافأة عيد ميلاد/ذكرى انضمام (نقاط أو مشروب مجاني).
15. هدية فورية + تهنئة عند ترقية المستوى.
16. سياسة اختيارية لانتهاء النقاط بعد خمول لتحفيز التكرار.
17. إحالة صديق: نقاط عند جلب عميل جديد عبر كود إحالة.
18. مضاعفة نقاط في أيام/أوقات محددة (Happy hour ولاء).
19. حماية ضد التلاعب: منع تكرار المسح لنفس الطلب، سقف يومي، ربط بالجوال الموحّد (phoneId).
20. لوحة تحليلات الولاء: نشِط/خامل، تكرار، قيمة العميل الدائمة (LTV)، أثر الخصومات.
