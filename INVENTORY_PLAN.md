# خطة المخزون الحقيقي (مواد خام + وصفات BOM + تحويل وحدات) — MenuLink

> نعم، أفهم النموذج: الكافيه يشتري **مواد خام** (بن، سكر، شاي، زنجبيل، حليب…) بكميات (كيلو/لتر/قطعة)، ولكل **صنف وحجم** وصفة تستهلك جراماً/مللياً محدّداً من كل مادة (قهوة صغيرة = 12غ بن، وسط = 15، كبير = 18). عند البيع يُخصم الاستهلاك من المواد الخام تلقائياً عبر **تحويل وحدات دقيق**.

ما بنيناه حالياً = «مخزون بسيط» (عدّ المنتج النهائي) — يبقى **كوضع Simple** للأصناف الجاهزة (ماء معبّأ، معجنات تُشترى جاهزة). نضيف فوقه **وضع Recipe** (المواد الخام + الوصفات).

---

## 1) الوحدات والتحويل

عائلات وحدات بوحدة أساس واحدة لكل عائلة:
| العائلة | الأساس | أمثلة تحويل |
|--------|-------|------------|
| كتلة | جرام (g) | kg=1000g |
| حجم | مل (ml) | L=1000ml |
| عدد | قطعة | dozen=12، box=N (مخصّص) |

- كل مادة لها `baseUnit` (تُخزّن به الكمية) و`purchaseUnit` + `purchaseFactor` (للاستلام): مثلاً بن baseUnit=g، purchaseUnit=kg، factor=1000.
- **تحويلات مخصّصة لكل مادة** عند الحاجة: 1 قطعة خبز = 40g، 1 شوت = 30ml.
- دالة `toBase(qty, unit, material)` / `fromBase(...)` تتولّى التحويل.

## 2) نموذج البيانات

```
tenants/{tid}/materials/{id}     // مادة خام
  { nameAr, nameEn, baseUnit:'g'|'ml'|'pc', stockQty:Number(base),
    reorderLevel:Number, avgCost:Number(per base), category, supplierId,
    purchaseUnit, purchaseFactor, expiryTracked, active }

tenants/{tid}/items/{id}         // إضافة على الصنف الحالي
  stockMode: 'none'|'simple'|'recipe'
  recipe:        [{ materialId, qty }]            // qty بوحدة الأساس (الافتراضي/بدون حجم)
  variantRecipes:{ [variantKey]: [{ materialId, qty }] }   // وصفة لكل حجم
  // الإضافات: لكل خيار modifier اختيارياً recipe:[{materialId, qty}]

tenants/{tid}/stockMoves/{id}    // حركة مخزون (سجل كامل)
  { type:'receive'|'sale'|'waste'|'count'|'transfer', materialId,
    qty:±base, cost?, orderId?, reason?, byName, at }

tenants/{tid}/suppliers/{id}     // مورّد (اختياري)
tenants/{tid}/locations/{id}     // فرع/مخزن (للتعدّد لاحقاً)
```

## 3) محرّك الاستهلاك (الخصم عند البيع)

عند تحوّل الطلب إلى **مدفوع** (مكان الكسب نفسه):
1. لكل بند: `recipe = variantRecipes[variantKey] ?? recipe`.
2. لكل `{materialId, qty}`: اخصم `qty × lineQty` من `materials[materialId].stockQty` عبر `increment(-x)` داخل معاملة، وسجّل `stockMove(type:'sale', orderId)`.
3. أضِف استهلاك الإضافات (modifier recipes).
4. لو نقص ≤ `reorderLevel` → تنبيه نقص.

> التزامن: استخدام `increment` + معاملات يمنع تضارب الخصم من طلبات متزامنة.

## 4) الاستلام والتكلفة (COGS)

- **استلام بضاعة**: اختر المادة + الكمية بوحدة الشراء (kg) → تحويل للأساس → `stockQty += `، وتحديث **متوسط التكلفة المرجّح** + `stockMove(type:'receive', cost)`.
- **تكلفة الصنف** = Σ(qty المادة × avgCost) ← هامش الربح لكل صنف وتقرير COGS.

## 5) الجرد والهدر

- **جرد فعلي (count)**: إدخال العدّ الفعلي → فرق (نظري − فعلي) → تسوية + `stockMove(type:'count')` + تقرير انحراف.
- **هدر (waste)**: خصم بسبب (تلف/انتهاء) + `stockMove(type:'waste')`.
- **صلاحية**: تتبّع تاريخ + تنبيه قرب الانتهاء.

## 6) الأسطح

- **قسم المخزون** (مطوّر): تبويبات «المواد الخام / الوصفات / الاستلام / الحركات / الجرد / التقارير».
- **بطاقة الصنف**: محرّر الوصفة لكل حجم + الإضافات.
- **تنبيهات**: نقص/انتهاء في لوحة المدير.

## 7) المراحل

1. المواد الخام CRUD + الوحدات/التحويل + الاستلام + متوسط التكلفة.
2. محرّر الوصفات لكل صنف/حجم/إضافة.
3. محرّك الاستهلاك عند الدفع + سجل الحركات (stockMoves).
4. تنبيهات النقص + إعادة الطلب + الموردون.
5. الجرد/الانحراف + الهدر + COGS وهامش الربح.
6. تعدّد المخازن/الفروع + التحويل بينها (لاحقاً).

## 8) قواعد Firestore (تُضاف)
`materials` / `stockMoves` / `suppliers` / `locations`: قراءة وكتابة للموظفين المخوّلين (`MANAGE_MENU`/مدير)، الحذف للمدير. لا قراءة عامة.

## 9) التوافق
- يبقى `stockMode:'simple'` (ما بنيناه) للأصناف الجاهزة.
- `stockMode:'recipe'` للأصناف المُحضّرة. الصنف بلا تتبّع = `'none'`.
