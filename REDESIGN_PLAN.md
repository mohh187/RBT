# خطة التحوّل الكبرى — MenuLink/كبدي (نسخة 2026-06-28)

تحوّل شامل لجعل كل واجهة في النظام احترافية وعملية وواقعية وكثيفة بالشكل الصحيح، **دون المساس بالبنية التحتية** (نفس المكوّنات/قاعدة البيانات/المعمارية). كل خطوة تُبنى تدريجياً مع تحقّق بناء نظيف.

---

## 1) خلاصة الفحص (٥ وكلاء)

**أ. هيكل لوحة الإدارة (AdminLayout):** تنقّل مسطّح — ٤ أزرار سفلية + ورقة «المزيد» فيها ١٥ رابطاً بلا تجميع. النمط الوحيد الصحيح للأقسام الرئيسية/الفرعية هو **StaffHub** (`/admin/hr`) بتبويبات محفوظة في الـ URL. مشاكل: نقاط دخول مكرّرة (صفحات الموظفين تُفتح من الهَب ومن روابط مستقلة)، وخلط صفحات الإدارة مع أسطح التشغيل (كاشير/مطبخ/بوابة/مسح).

**ب. الكاشير/المطبخ:** ليست POS حقيقية بل لوحات تتبّع طلبات. الكاشير = لوحة كانبان (جديد→تحضير→جاهز) بالسحب. المطبخ = شبكة بطاقات. ينقص: إدخال طلب من الموظف، دفع، بحث، تبويبات أقسام، ممرات حالة، كثافة، أهداف لمس كبيرة، إبراز الإضافات، مؤقّتات. كثافة منخفضة (٤–٦ تذاكر/شاشة).

**ج. صفحات الإدارة (٢٢ صفحة):** كبيرة جداً — عناوين 28px، أرقام إحصاء 28px، حقول 44px، صور 84px، مناطق نص 84px. تباعد منتفخ (16px). الكثافة غير متّسقة. صفحة الإعدادات هي المنطقة الكثيفة الوحيدة (تجاوزات `.settings-page`). لا نظام كثافة منهجي.

**د. نظام التصميم/الثيمات:** السلّم الكامل موثّق (أساس 15px لكن المكوّنات تضخّمه). بنية الثيمات موجودة (skins/applySkin/resolveSkin/surfaceThemes/applyTypography) لكن: **أسطح الإدارة لا تستدعي applySkin إطلاقاً** (بلا ثيم/خط)، تغيير الخط **لا يُحفظ**، و`surfaceThemes` **مُشار إليه لكنه غير مُفعّل**. ثيم-لكل-سطح مُصمّم لكنه غير موصول.

**هـ. المصادقة/الباقات/البوابة:** التسجيل **يجمع `type` أصلاً** (cafe/restaurant/other) لكن لا يستخدمه للثيم/المزايا. الباقات (plans.js): ٤ مستويات (menu/ops/pro/enterprise) + خريطة `FEATURE_MIN` (~٢٠ مفتاح) + `planAllows()`، الافتراضي «enterprise» (لا شيء مقفل). يُستخدم فقط لفحص «الثيمات». نظام الصلاحيات (CAP) منفصل. بوابة الموظف (٧٦٧ سطر) غنية ومستقلّة.

---

## 2) النظام: أقسام رئيسية + فرعية (مثل شؤون الموظفين)

التنقّل الجديد = **٦ هَبات رئيسية**، كلٌّ بتبويبات فرعية (نمط StaffHub، محفوظة في الـ URL):

1. **لوحة القيادة** — نظرة اليوم · التقرير اليومي · التقارير.
2. **المنيو** — الأصناف · التصنيفات · العروض.
3. **العمليات** — الطلبات/الكاشير · المطبخ · الطاولات · الحجوزات · الفعاليات.
4. **العملاء** — العملاء · الشكاوى · الولاء.
5. **الفريق** — (StaffHub الحالي: الأعضاء/الأداء/الجدول/الحضور/الإجازات/الإعلانات) + الأدوار + السياسات.
6. **الإعدادات** — البيانات · المظهر · الثيمات/سطح · التشغيل · الرابط · الباقة/الفوترة · التطبيق.

شريط سفلي = الهَبات الـ٥–٦ + شريط فرعي لكل هَب. إزالة نقاط الدخول المكرّرة. على الكمبيوتر: شريط جانبي قابل للطيّ؛ على الجوال: شريط سفلي.

## 3) المراحل (تدريجية، غير مُخرّبة)

### المرحلة 0 — نظام الكثافة والأحجام («كبير جداً» يُحَلّ جذرياً)
- إضافة سلّم كثافة `[data-density='compact']` (مجموعة توكنات موازية: ‎--fs/--sp/--tap/--appbar-h‎ مصغّرة).
- وضع `data-density='compact'` على غلاف الإدارة + أسطح التشغيل؛ يبقى منيو الزبون «مريحاً».
- خفض منهجي: عناوين 28→20px، أرقام 28→18px، حقول 44→36px، حشوات/فجوات. **كل الصفحات تصبح كثيفة دفعةً واحدة عبر الـ cascade** دون لمس كل صفحة.
- توحيد البطاقة/الصف/العنوان في مواصفة واحدة.

### المرحلة 1 — هيكلة لوحة الإدارة (هَبات + تبويبات فرعية)
- بناء غلاف إدارة احترافي: شريط علوي (شعار + بحث ⌘K + إجراءات + ملف)، شريط جانبي (كمبيوتر) / سفلي (جوال) بالهَبات الـ٦.
- لكل هَب: شريط تبويبات فرعية لاصق (نمط StaffHub، URL-persisted). توجيه الصفحات المستقلّة عبر الهَبات.
- نمط رأس صفحة موحّد: فتات الخبز + عنوان + صف إجراءات.

### المرحلة 2 — أسطح التشغيل (كاشير + مطبخ احترافيان)
- **الكاشير:** لوحة كثيفة + ممرات حالة + تبديل عرض (لوحة/قائمة) + بحث/فلترة + أهداف لمس كبيرة + (اختياري) **وضع POS**: تبويبات أقسام + شبكة أصناف + لوح تذكرة + دفع سريع لإدخال الطلب من الموظف.
- **المطبخ:** ممرات (جديد/تحضير/جاهز) + أزرار «تم» كبيرة + إبراز الإضافات بوضوح + مؤقّتات SLA ملوّنة + فلترة محطّات + وضع كثيف + داكن افتراضي.

### المرحلة 3 — محرّك الثيمات v2 (خط عام + ثيم لكل سطح)
- وصل applySkin عالمياً (admin/cashier/kds/portal — حالياً بلا ثيم).
- **حفظ الخط** (تبديل خط النظام بالكامل اختيارياً، منفصل عن الثيم).
- **ثيم لكل سطح** (menu/dashboard/cashier/kitchen/portal) + «طبّق على الكل» (البنية موجودة — نوصلها + واجهة في الإعدادات).
- مبدّل ثيمات احترافي بمعاينة حيّة لكل سطح.

### المرحلة 4 — التسجيل/الدخول + نوع المنشأة → ثيم
- إعادة تصميم الدخول/التسجيل/التهيئة (احترافي، متعدّد الخطوات بمؤشّر تقدّم، أحجام حقول مناسبة).
- **نوع المنشأة** (كافيه/مطعم/محل/مخبز/عصائر/حلويات…) → **توصيات ثيمات منسّقة** لكل نشاط.
- تخطيط دخول احترافي (لوحة علامة + نموذج).

### المرحلة 5 — الباقات والمزايا (تقسيم + بوابة فتح نظيفة)
- مصفوفة المزايا الـ٥٠ على المستويات الأربعة.
- مكوّن `<FeatureGate>` نظيف: الميزة المقفلة تظهر بقفل لطيف + تلميح ترقية (بلا كسر للواجهة).
- صفحة الباقة/الفوترة: بطاقات المستويات + مصفوفة المزايا + زر ترقية.

### المرحلة 6 — تلميع بوابة الموظف + ثيمها الخاص
- تنظيف الصفحة الرئيسية (أقسام قابلة للطيّ) + تبويبات حسب المستوى + سطح ثيم «portal».

## 4) ٣٠+ اقتراح واجهة/تصميم

1. نظام كثافة `data-density` (compact للإدارة، comfortable للزبون).
2. عناوين أصغر + فتات خبز بدل العناوين الضخمة.
3. غلاف إدارة متجاوب: شريط جانبي (كمبيوتر) + سفلي (جوال).
4. هَبات رئيسية بتبويبات فرعية في كل مكان (نمط StaffHub).
5. شريط تبويبات فرعي لاصق بمؤشّر نشِط.
6. مواصفة بطاقة واحدة موحّدة (حشوة واعية بالكثافة).
7. بطاقات KPI مدمجة بخط بياني صغير، أرقام 18px.
8. **جداول بيانات** للقوائم على الكمبيوتر بدل البطاقات المكدّسة (أكثف).
9. مواصفة صف موحّدة (أيقونة + نصّان + ميتا + إجراء).
10. حالات فارغة احترافية (أيقونة + عنوان + إجراء، بلا إيموجي).
11. درج تفاصيل جانبي على الكمبيوتر بدل الأوراق السفلية.
12. لوحة أوامر ⌘K للتنقّل/البحث السريع.
13. شريط علوي عام: بحث + إجراءات سريعة + قائمة ملف.
14. نمط رأس صفحة موحّد (عنوان + صف إجراءات).
15. وضع POS للكاشير: تبويبات أقسام + شبكة أصناف + لوح تذكرة + لوحة دفع.
16. لوحة كاشير مدمجة + ممرات حالة + مبدّل كثافة.
17. ممرات مطبخ (جديد/طبخ/جاهز) + أزرار «تم» كبيرة + إبراز الإضافات + مؤقّتات.
18. مطبخ داكن افتراضي + فلترة محطّات.
19. ألوان استعجال موحّدة (مؤقّتات SLA) عبر الكاشير/المطبخ.
20. تفاصيل طلب: مخطّط زمني + أصناف + دفع في لوح نظيف.
21. نظام نماذج: عمودان على الكمبيوتر، أقسام حقول مجمّعة، حقول 36px.
22. تحقّق فوري + نص مساعد + أدوات مقسّمة للقيم المعدودة.
23. مبدّل ثيمات: معرض + معاينة حيّة + تبويبات لكل سطح + «طبّق على الكل».
24. مبدّل خط النظام (منفصل عن الثيم) بمعاينة.
25. معالج تهيئة متعدّد الخطوات (نوع → بيانات → ثيم → تمّ) بمؤشّر تقدّم.
26. توصيات ثيمات حسب نوع النشاط.
27. دخول/تسجيل بتخطيط احترافي (لوحة علامة + نموذج).
28. صفحة باقة/فوترة: بطاقات مستويات + مصفوفة مزايا + ترقية.
29. `FeatureGate`: الميزة المقفلة بقفل لطيف + تلميح ترقية.
30. توستات/حوارات متّسقة احترافية (بلا إيموجي) + تراجع.
31. هياكل تحميل (skeletons) بدل الدوّارات.
32. لوحة قيادة بعناصر حسب الدور + نبض اليوم + تنبيهات.
33. بوابة موظف أنظف (أقسام قابلة للطيّ) + تبويبات حسب المستوى.
34. أيقونات موحّدة + تنقّل أيقوني بتلميحات على الكمبيوتر.
35. إعدادات مُعاد تنظيمها لتبويبات فرعية واضحة.

## 5) ٥٠ ميزة موزّعة على الباقات

### الباقة 1 — منيو (Menu)
1. منيو QR (طاولة + رابط عام) · 2. ثنائي اللغة AR/EN · 3. تصنيفات وأصناف (صور/أحجام/إضافات/مكوّنات) · 4. هوية أساسية (شعار/غلاف/ألوان) · 5. تبديل توفّر الصنف · 6. بحث في المنيو · 7. عرض التقييمات · 8. ثيمات أساسية (٣–٤) · 9. معاينة منيو حيّة · 10. اختيار العملة.

### الباقة 2 — تشغيل (Operations)
11. طلب QR (سلة + دفع) · 12. لوحة طلبات (كاشير) · 13. شاشة مطبخ (KDS) · 14. حالات وتتبّع حيّ · 15. طاولات + QR لكل طاولة · 16. أنواع طلب (محلي/استلام/سيارة) · 17. حجوزات · 18. فعاليات + تذاكر · 19. عروض وخصومات (سلة/تصنيف/صنف، محدّدة المدة) · 20. متصفّح عروض + تطبيق تلقائي · 21. نداء النادل · 22. إشعارات طلبات (push) · 23. تسجيل عملاء (CRM خفيف) · 24. طباعة الطلبات.

### الباقة 3 — احترافي (Pro)
25. مكتبة ثيمات (٣٠+) · 26. تخصيص عميق لكل ثيم (ألوان/خطوط/شكل/تخطيط) · 27. ثيمات مخصّصة بأسماء (حفظ/تبديل) · 28. بانر + خلفيات + فيديو + علامة مائية · 29. شاشات صنف عامرة + صور متعددة + منتجات شفافة · 30. مكتبة حركات الأصناف + تفاعلات لمس · 31. أنماط هيدر/قائمة سفلية/تبويبات أقسام · 32. تحكّم خطوط دقيق (لون/حجم لكل عنصر) · 33. **ثيم لكل سطح** · 34. **خط نظام عام** · 35. برنامج ولاء · 36. تقارير وتحليلات متقدّمة · 37. تحليلات عملاء + تصدير CSV · 38. كثافة/خط مخصّص للوحة.

### الباقة 4 — متكامل (Enterprise)
39. إدارة موظفين (دليل/أدوار/صلاحيات) · 40. حضور + جيوفنس + بصمة بصورة · 41. أداء + أهداف + لوحة صدارة · 42. مناوبات وجدولة + تبديل · 43. إجازات · 44. رواتب (راتب/خصومات/إضافي/مكافآت) · 45. إعلانات · 46. بوابة موظف كاملة · 47. تقرير يومي · 48. سياسات (حضور/استراحة/إجازة/إضافي/مكافآت) · 49. شكاوى · 50. لوحة تحكّم مركزية (سوبر-أدمن) + تعدّد فروع + دومين مخصّص (مستقبلي).

(كل مستوى أعلى يشمل ما دونه. الافتراضي يبقى مفتوحاً حتى نُفعّل التقسيم.)

## 6) مبدأ التنفيذ
- لا حذف/كسر للبنية. كل مرحلة = إضافات + إعادة تنظيم بالـ cascade.
- بناء مُتحقَّق بعد كل خطوة. تقسيم كل مرحلة لخطوات صغيرة.
- البداية الموصى بها: **المرحلة 0 (الكثافة)** — أكبر أثر بصري، أقل خطر، تُحسّن كل الصفحات فوراً.
