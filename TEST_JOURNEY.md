# رحلة اختبار الدفع الكاملة — rbt360 (وضع Moyasar التجريبي)

هدف هذا الملف: تشغيل رحلة كاملة (طلب → دفع → تحضير → إغلاق) + اشتراك + تذكرة + عربون،
باستخدام **مفاتيح Moyasar التجريبية** (لا رسوم حقيقية). المفاتيح الحالية في المشروع
حيّة (`sk_live`/`pk_live`)، وحساب التاجر غير مُفعّل بعد — لذا **الوضع التجريبي هو
الطريق الوحيد لاختبار الدفع الآن**.

> بطاقات Moyasar التجريبية: نجاح `4111 1111 1111 1111` (Visa) — أي تاريخ مستقبلي، أي CVC،
> أي اسم؛ في صفحة 3-D Secure اتبع رمز الاختبار المعروض. للرفض استخدم بطاقة الرفض من
> [توثيق Moyasar](https://docs.moyasar.com/testing).

---

## 0) المفاتيح التجريبية — تم ضبطها (مؤكَّد)

مفاتيح Moyasar التجريبية مضبوطة الآن في `functions/.env` و`.env.local` (git-ignored، والمفاتيح الحيّة
محفوظة في أسطر معلّقة للاستعادة)، و`PUBLIC_BASE_URL` مضبوط. تم التحقّق: إنشاء فاتورة تجريبية نجح.

للإيميل/الواتساب (اختياري لتجربتهما): املأ في `functions/.env` قيم `RESEND_API_KEY` + نطاق
`EMAIL_FROM` موثّق، و`WA_PHONE_NUMBER_ID` + `WA_ACCESS_TOKEN` + قالب Meta معتمد `order_update`.

## 1) تحقّق سريع من طبقة الدفع (بلا نشر)

بعد وضع `sk_test_` في `functions/.env`:

```bash
node scripts/moyasar-test-journey.mjs 10
```

- يُنشئ فاتورة Moyasar تجريبية بـ 10 ريال، يطبع رابط الدفع.
- افتح الرابط، ادفع ببطاقة تجريبية، وسيطبع السكربت الحالة حتى `paid` وتفاصيل الدفعة.
- هذا يُثبت أن `createInvoice` + جلب الحالة + جلب الدفعة تعمل بالمفتاح — وهي نفس نداءات
  `createPayIntent` / `settleFromPayment` / الويبهوك. (السكربت يرفض العمل بمفتاح حيّ.)

## 2) الرحلة الكاملة (نشر مؤقّت بمفاتيح تجريبية)

> يتطلّب أن تنشر أنت (لا أملك صلاحية menu-88996). انشر **الدوال ثم القواعد** (لأن التذاكر
> المجانية صارت تمرّ عبر `issueFreeTicket`).

### 2.1 البناء والنشر
```bash
npm run check          # guard + eslint + build (أخضر)
firebase deploy --only functions          # يبعث المفاتيح التجريبية (تُخبز عند النشر)
firebase deploy --only firestore:rules,storage
firebase deploy --only hosting
```

### 2.2 ربط الويبهوك (وضع Moyasar التجريبي)
- في لوحة Moyasar (Test mode) → Webhooks → أضف:
  - URL: `https://<region>-menu-88996.cloudfunctions.net/paymentWebhook` (أو رابط الدالة v2).
  - Secret token: نفس `MOYASAR_WEBHOOK_SECRET` في `functions/.env`.
  - الأحداث: `payment_paid` (وما يلزم).
- الويبهوك يفشل مغلقاً بلا سرّ صحيح؛ ومسار `/pay/return` يسوّي أيضاً (حزام ومشبك) فلا
  تعتمد الرحلة على الويبهوك وحده.

### 2.3 تجهيز منشأة تجريبية
- سجّل منشأة (أو استخدم `mazagfal`)، أضف فئة + أصنافاً بأسعار.
- فعّل الدفع الإلكتروني: `tenants/{tid}.onlinePayment.enabled = true` (من الإعدادات).

### 2.4 الطلب + الدفع (المسار الحرِج)
1. افتح `/m/{slug}` كزائر، أضف أصنافاً، فعّل «الدفع الإلكتروني»، اطلب.
2. تُحوَّل لصفحة Moyasar → ادفع ببطاقة `4111 1111 1111 1111`.
3. العودة إلى `/pay/return` → «تم الدفع».
4. **تحقّق:**
   - `payIntents/{id}.status == 'paid'` مرّة واحدة (idempotent — أعِد تحميل `/pay/return`: لا تكرار).
   - الطلب يبقى `pending` ويظهر **في KDS ويُحضَّر** (لم يُغلق).
   - في الكاشير: افتح الطلب → PaymentSheet «المطلوب = 0» + شارة «مدفوع أونلاين — لا تُحصّل نقداً».
   - تتبّع العميل يُظهر «مدفوع».
5. أغلق الطلب من الكاشير (اختر «شبكة/أونلاين») → تعمل تأثيرات ما بعد الدفع (نقاط الولاء/خصم المواد).

### 2.5 توصيل مدفوع أونلاين
- اطلب توصيلاً وادفع أونلاين، ثم من `/driver` سلّمه → **لا مطالبة COD**، يُوسم delivered فقط.

### 2.6 اشتراك المنشأة (المنشأة تدفع للمنصة)
- من `/admin` بطاقة الاشتراك، ادفع فاتورة `unpaid` → بعد الدفع: الفاتورة `paid`، تُمدَّد الباقة.
- (الآن يتطلّب مديراً مسجّلاً — R2.)

### 2.7 تذكرة مدفوعة + مجانية
- **مدفوعة:** احجز نوعاً مدفوعاً → `pending` → ادفع → `valid`. المبلغ = سعر الفعالية (لا يتأثّر بتلاعب العميل).
- **مجانية:** احجز نوعاً بسعر 0 → تُصدَر عبر `issueFreeTicket` كـ `valid` (الخادم يتحقّق أنها فعلاً 0).
- امسحها من `/admin` الماسح → `used`.

### 2.8 عربون حجز
- احجز مناسبة تتطلّب عرباناً → ادفع → الحجز `confirmed`, `depositStatus:'paid'`.

## 4) الاستعادة بعد الاختبار
- أعِد المفاتيح الحيّة في `functions/.env` + `.env.local`.
- `firebase deploy --only functions,hosting` (المفاتيح تُخبز عند النشر — لا بدّ من إعادة النشر).
- في لوحة Moyasar، بدّل من Test إلى Live، وأعِد تسجيل الويبهوك على الوضع الحيّ.
- **بدّل كلمة المرور (Rashash-18) وPIN (1234)** المكشوفَين سابقاً.

### ملاحظات
- الدوال v2 **تخبز `.env` عند النشر** — أي تغيير مفتاح يتطلّب إعادة نشر الدوال (درس من wbs).
- التذاكر المجانية: انشر الدوال + الاستضافة **قبل** القواعد (وإلا تُرفض تذكرة الزائر المجانية مؤقّتاً).
- بديل بلا نشر لكامل الرحلة: Firebase Emulator Suite (يتطلّب Java + ربط العميل بالمحاكيات + بذر منشأة)؛ الويبهوك لا يصل المحاكي، لكن مسار `/pay/return` يسوّي عبر Moyasar API مباشرة.
