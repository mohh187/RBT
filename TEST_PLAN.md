## 1) التسجيل والدخول والأدوار

**1.1 تسجيل مالك جديد + إنشاء منشأة**
- الخطوات: افتح `/signup` → أنشئ حساباً → أكمل onboarding (اسم المنشأة، الـ slug، الشعار).
- المتوقّع: ✅ يُنشأ `users/{uid}` بدور `owner`، `tenants/{tid}` مع `ownerUid==uid`، و`tenantSlugs/{slug}` فريد. تحويل للوحة `/admin`.

**1.2 slug مكرّر** — حاول إنشاء منشأة بـ slug مستخدم. المتوقّع: ✅ رفض (uniqueness عبر doc id).

**1.3 دعوة موظّف** — من `/admin/staff` أنشئ دعوة بالبريد + الدور. سجّل بذلك البريد في نافذة أخرى.
- المتوقّع: ✅ الموظّف يطالب دوره من `staffInvites`، يُربط بنفس `tenantId`، ولا يستطيع أخذ دور أعلى من الدعوة.

**1.4 قفل PIN التشغيلي** — فعّل PIN، أقفل، أعد الفتح بـ PIN صحيح/خاطئ.
- المتوقّع: ✅ الصحيح يفتح؛ الخاطئ يُرفض؛ يُخزَّن hash فقط لا plaintext.

**1.5 الصلاحيات (caps)** — سجّل بدور `cashier` وحاول فتح `/admin/staff` أو `/admin/reports`.
- المتوقّع: 🔒 `RequireCap` يمنع؛ لا يظهر العنصر في القائمة.

## 2) المنيو العام والطلب

**2.1 تصفّح المنيو** — افتح `/m/{slug}` كزائر.
- المتوقّع: ✅ يحمّل خلال ثوانٍ (لا سبينر أبدي)، الفئات والأصناف والعروض تظهر، السمة/الشعار مطبّقان فوراً (cache).

**2.2 فشل التحميل** — افتح slug غير موجود، ثم slug صحيح مع قطع الشبكة.
- المتوقّع: ✅ الأول «المنشأة غير موجودة»؛ الثاني حالة خطأ «حدّث الصفحة» (بعد watchdog 8ث كحدّ أقصى) لا سبينر أبدي.

**2.3 السلة والطلب (dine-in)** — امسح QR طاولة → أضف أصنافاً بخيارات/ملاحظات → أرسل الطلب.
- المتوقّع: ✅ يُنشأ `orders/{id}` بحالة `pending`، `statusHistory:[{pending}]`، مربوط بالطاولة، يظهر في الكاشير/KDS.

**2.4 تقييد الطلبات بالنطاق (geofencing)** — من خارج إحداثيات المنشأة، جرّب طلب طاولة.
- المتوقّع: ✅ طلبات الطاولات مقيّدة بحرم المنشأة؛ المنيو العام يسمح بالسفري/التوصيل. التوصيل مقيّد بالمسافة القصوى.

**2.5 تتبّع الطلب** — بعد الإرسال افتح رابط `/order/{slug}/{id}` (أو زر التتبّع).
- المتوقّع: ✅ خطوات الحالة تتقدّم حيّاً؛ لطلب توصيل تظهر مسافة/ETA المندوب.

## 3) الكاشير ونقطة البيع

**3.1 فتح/إغلاق جلسة درج نقدي** — افتح جلسة برصيد افتتاحي، أغلقها.
- المتوقّع: ✅ تُحتسب المبيعات النقدية للجلسة والموظّف المسجَّل فقط.

**3.2 دفع طلب — نقدي/شبكة/تحويل** — من الكاشير افتح طلباً `ready`/`served` → PaymentSheet → اختر الطريقة → أكّد.
- المتوقّع: ✅ الحالة → `paid`، تُسجَّل الطريقة والموظّف والوقت؛ يظهر في «المنجزة».

**3.3 دفع مختلط** — اختر «مختلط» ووزّع نقدي/شبكة/تحويل. المتوقّع: ✅ يُرفض الزرّ لو المجموع أقل من المطلوب؛ يُقبل عند التغطية.

**3.4 دفعة جزئية + تقسيم** — سجّل مبلغاً أقل من المطلوب / قسّم على عدد أشخاص.
- المتوقّع: ✅ يبقى الطلب «جزئي · متبقّي X»، ويُغلق تلقائياً حين يغطّي التراكم الإجمالي.

**3.5 إكرامية** — أضف إكرامية ثابتة/مخصّصة. المتوقّع: ✅ تُضاف للمطلوب وتُسجَّل.

**3.6 مجاملة/خصم واسترجاع** — طبّق comp على طلب نشط؛ استرجع طلباً مدفوعاً (مدير).
- المتوقّع: ✅ الخصم يظهر؛ الاسترجاع يسجّل المبلغ/السبب/المنفّذ ويقلّل الإيراد.

## 4) الدفع الإلكتروني (الحرِج — راجع AUDIT_DEEP)

> المتطلّب: `tenant.onlinePayment.enabled==true`، مفاتيح Moyasar مضبوطة، الدوال منشورة.

**4.1 طلب مدفوع أونلاين (منيو)** — كزائر أضف أصنافاً، فعّل «الدفع الإلكتروني»، اطلب → صفحة Moyasar → ادفع ببطاقة تجريبية → العودة إلى `/pay/return`.
- المتوقّع: ✅ `payIntents/{id}` → `paid`؛ الطلب يبقى `pending` ويظهر **في KDS ويُحضَّر** (لا يُغلق)؛ `paidOnline:true, amountPaid=total`؛ العميل يرى «مدفوع» وإشعاراً.
- 🔒 كرّر العودة/الويبهوك مرّتين: لا تمديد/تفعيل مزدوج (idempotent).

**4.2 عدم تحصيل مزدوج** — بعد 4.1 افتح الطلب في الكاشير.
- المتوقّع: ✅ PaymentSheet «المطلوب = 0»، شارة «مدفوع أونلاين — لا تُحصّل نقداً».

**4.3 توصيل مدفوع أونلاين** — اطلب توصيلاً وادفع أونلاين، ثم من `/driver` سلّمه.
- المتوقّع: ✅ لا مطالبة COD؛ يُوسم delivered فقط.

**4.4 دفع اشتراك المنشأة (المنشأة تدفع للمنصة)** — من `/admin` بطاقة الاشتراك، ادفع فاتورة `unpaid`.
- المتوقّع: ✅ الفاتورة → `paid`، تُمدَّد الباقة (سنوي 365 / شهري 30) ويُعاد التفعيل.

**4.5 تذكرة مدفوعة** — احجز تذكرة نوع مدفوع، ادفع.
- المتوقّع: ✅ التذكرة `pending` → `valid` بعد الدفع؛ المبلغ = **سعر نوع التذكرة في الفعالية** حتى لو تلاعب العميل.

**4.6 عربون حجز** — احجز مناسبة تتطلّب عربوناً، ادفع.
- المتوقّع: ✅ الحجز `requested`→`confirmed`، `depositStatus:'paid'` بعد الدفع فقط.

**4.7 اختبارات سلبية للويبهوك** — راجع سيناريوهات AUDIT_DEEP 1,2,7,8,9. المتوقّع: 🔒 كلها تُرفض/لا تُسوّي.

## 20) فحوص شاملة (كل شاشة)

- **RTL** — كل الشاشات بالعربية RTL سليمة، لا كسر تخطيط.
- **الأرقام** — لا أرقام هندية عربية إطلاقاً؛ Latin digits فقط (`ar-SA-u-nu-latn`). شغّل `npm run guard`.
- **الأيقونات** — لا إيموجي/رموز نصّية؛ أيقونات فقط (`Icon`/SVG). `guard` أخضر.
- **التمرير** — لا أشرطة تمرير مرئية؛ overlay hover فقط.
- **i18n** — بدّل ar/en على عيّنة شاشات؛ لا نصوص مفقودة.
- **الوضع الداكن** — عيّنة شاشات + المنيو + السمات تعمل داكنةً.
- **الأداء** — المنيو والشاشات الثقيلة تحمّل بلا تعليق؛ كل `onSnapshot` له callback خطأ (لا سبينر أبدي).
- **البناء** — `npm run check` (guard + eslint + vite) أخضر قبل كل إصدار.
