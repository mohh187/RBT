## 1. المهارات المثبّتة — مجموعات وظيفية

### أ. المحرّك الرئيسي للتصميم/إعادة التصميم
- **`impeccable`** (الأقوى، v3.9.1): تصميم/إعادة تصميم/تدقيق/صقل واجهات إنتاجية. له **أوامر فرعية**: `craft` (بناء جديد)، `shape` (تشكيل)، `audit` (تدقيق)، `polish` (صقل). عند تشغيله يقرأ `PRODUCT.md`/`DESIGN.md` ويشغّل `scripts/context.mjs` — إن لم يوجد PRODUCT.md يتّبع `reference/init.md` لإنشائه. **هذا محرّكنا لكل شاشة.**
- **`redesign-existing-projects`**: يرقّي مشروعاً قائماً — يدقّق، يكشف أنماط «الذكاء الاصطناعي العامة»، ويطبّق معايير راقية **دون كسر الوظائف** (لا يعيد الكتابة من الصفر). ← مثالي لنا لأن النظام قائم.
- **`design-taste-frontend`** (+ `design-taste-frontend-v1`): واجهات «غير قوالبية»، تدقيق-أولاً على إعادة التصميم، فحص قبلي صارم.
- **`high-end-visual-design`**: يجعل الواجهة تبدو «باهظة» — خطوط/مسافات/ظلال/بطاقات/حركة الوكالات الراقية، ويحظر الافتراضات الرخيصة.

### ب. الصقل والحركة (تفاصيل الاحتراف)
- **`emil-design-eng`**: فلسفة Emil Kowalski في صقل الواجهة والحركة والتفاصيل الخفية التي تجعل البرنامج «يُحَسّ» رائعاً.
- **`review-animations`**: يراجع كود الحركة بمعيار حِرفي عالٍ (يوسم افتراضياً؛ القبول يُكتسَب).
- **`animation-vocabulary`**: قاموس عكسي — يحوّل وصفاً غامضاً لحركة إلى مصطلحها الدقيق (لتوجيه الطلب بدقة).

### ج. توليد مراجع بصرية حقيقية (قبل الكود)
- **`imagegen-frontend-web`**: يولّد صورة مرجع منفصلة **لكل قسم** (موقع بـ8 أقسام = 8 صور) — تنوّع تركيب، أبطال متعددة الأحجام، لوحة موحّدة.
- **`imagegen-frontend-mobile`**: مراجع شاشات تطبيقات جوال داخل إطار هاتف.
- **`image-to-code`**: يولّد الصورة أولاً، يحلّلها بعمق، ثم يبني الواجهة لتطابقها.
- **`brandkit`**: لوحات هوية/شعارات/أنظمة بصرية راقية.
> ⚠️ هذه المهارات **تولّد صوراً** — تحتاج قدرة توليد صور في البيئة. إن لم تتوفّر، نستخدمها كـ**توجيه دقيق وبرومبتات** ننفّذها بمراجع خارجية (Dribbble/Mobbin/متاجر حقيقية).

### د. اتجاهات جمالية جاهزة (اختَر واحداً كهوية)
- **`minimalist-ui`**: تحريري نظيف، أحادي دافئ، بنتو مسطّح، بلا تدرّجات/ظلال ثقيلة.
- **`industrial-brutalist-ui`**: صناعي بروتالي، شبكات صارمة، تباين حجم هائل — يناسب **لوحات البيانات الكثيفة** (KDS/الكاشير).
- **`gpt-taste`**: تحريري + حركة GSAP متقدّمة (أنسب للصفحة التعريفية لا لـPOS الكثيف).
- **`stitch-design-taste`**: يولّد `DESIGN.md` دلالياً (لـ Google Stitch) بمعايير راقية.

### هـ. سلامة المخرجات
- **`full-output-enforcement`**: يمنع اقتطاع الكود ونماذج «placeholder»، ويضمن مخرجات كاملة. ← نفعّله في مهام الكود الطويلة.

## 2. سير العمل المتكامل للتحوّل الضخم (الحلقة لكل شاشة)

```
0) الأساس (مرة واحدة):
   • impeccable (init) → إنشاء PRODUCT.md + DESIGN.md (الهوية، اللوحة، الخطوط، القرارات).
   • اختيار اتجاه: high-end-visual-design + (minimalist-ui أو industrial-brutalist-ui للأقسام الكثيفة).
   • ترقية index.css (نظام التصميم) + مكوّنات الأساس.

1) لكل قسم (الكاشير → المطبخ → الداشبورد → …):
   أ. مرجع:   imagegen-frontend-web / image-to-code  (أو برومبت + مرجع خارجي).
   ب. توجيه:  design-taste-frontend + high-end-visual-design (القرارات).
   ج. بناء:   impeccable (craft/shape) + redesign-existing-projects + full-output-enforcement.
   د. حركة:   emil-design-eng (الصقل) ثم review-animations (المراجعة).
   هـ. تحقّق: npm run build + تشغيل/لقطة شاشة للتأكد بصرياً.

2) القوالب: نطبّق نفس الحلقة لكل «قالب» من قوالب القسم (data-template)، مع بوابة الباقة.
```

### كيف سأشغّلها عملياً
- عند تنفيذ قسم أقول مثلاً: «صمّم الكاشير» → تُحمّل `impeccable`/`redesign-existing-projects` تلقائياً، وأتّبع تدفّقها (PRODUCT.md، register=product، الفحص القبلي)، وأستدعي `emil-design-eng` للصقل، و`review-animations` للمراجعة.
- للمراجع البصرية أستدعي `imagegen-frontend-web` (إن توفّر توليد الصور) وإلا أستخدمها كتوجيه.
- أنت تكتب `/impeccable` أو `/redesign-existing-projects` أو `/polish` في أي وقت لتوجيه العمل.

## 3. نصائح لأفضل النتائج (من محتوى المهارات)
- **تباين نصّي ≥ 4.5:1** للنص الأساسي؛ لا رمادي فاتح «للأناقة» (أكبر سبب لقُبح تصاميم AI).
- **خطوط بشخصية** (Geist/Outfit/Satoshi للاتيني) + **أرقام tabular** للبيانات؛ الحفاظ على خط عربي راقٍ.
- **البطاقات هي الحل الكسول** — استخدمها عند الحاجة فقط، وممنوع بطاقة داخل بطاقة.
- **حركة مقصودة** ease-out أسّية، بلا bounce، مع `prefers-reduced-motion`.
- **كسر التماثل** والشبكات الثلاثية المتساوية (بصمة AI).
- **ظلال ملوّنة** بلون الخلفية لا أسود شفّاف.
- **RTL أولاً** (خاصّتنا) في كل قرار.
